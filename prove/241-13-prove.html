13 Prove : Assignment - Analyzing a Census

Overview

Now that we have discussed several different data structures and algorithms, we can apply those to solve interesting problems.

This week, your assignment is to write a program to analyze data from a census. This census data is from the 1994 census, and originally comes from the UCI Machine Learning Data Set Repository. A simplified version is provided in the Linux Lab: /home/cs241/census.txt

The census dataset consists of one record per line. The fields of each record are separated by spaces. For example:

39 State-gov 77516 Bachelors 13 Never-married Adm-clerical Not-in-family White Male 2174 0 40 United-States <=50K 50 Self-emp-not-inc 83311 Bachelors 13 Married-civ-spouse Exec-managerial Husband White Male 0 0 13 United-States <=50K 38 Private 215646 HS-grad 9 Divorced Handlers-cleaners Not-in-family White Male 0 0 40 United-States <=50K 53 Private 234721 11th 7 Married-civ-spouse Handlers-cleaners Husband Black Male 0 0 40 United-States <=50K

According to the dataset description page, the following are the fields of each record along with the possible values:

  • age: continuous.
  • workclass: Private, Self-emp-not-inc, Self-emp-inc, Federal-gov, Local-gov, State-gov, Without-pay, Never-worked.
  • fnlwgt: continuous.
  • education: Bachelors, Some-college, 11th, HS-grad, Prof-school, Assoc-acdm, Assoc-voc, 9th, 7th-8th, 12th, Masters, 1st-4th, 10th, Doctorate, 5th-6th, Preschool.
  • education-num: continuous.
  • marital-status: Married-civ-spouse, Divorced, Never-married, Separated, Widowed, Married-spouse-absent, Married-AF-spouse.
  • occupation: Tech-support, Craft-repair, Other-service, Sales, Exec-managerial, Prof-specialty, Handlers-cleaners, Machine-op-inspct, Adm-clerical, Farming-fishing, Transport-moving, Priv-house-serv, Protective-serv, Armed-Forces.
  • relationship: Wife, Own-child, Husband, Not-in-family, Other-relative, Unmarried.
  • race: White, Asian-Pac-Islander, Amer-Indian-Eskimo, Other, Black.
  • sex: Female, Male.
  • capital-gain: continuous.
  • capital-loss: continuous.
  • hours-per-week: continuous.
  • native-country: United-States, Cambodia, England, Puerto-Rico, Canada, Germany, Outlying-US(Guam-USVI-etc), India, Japan, Greece, South, China, Cuba, Iran, Honduras, Philippines, Italy, Poland, Jamaica, Vietnam, Mexico, Portugal, Ireland, France, Dominican-Republic, Laos, Ecuador, Taiwan, Haiti, Columbia, Hungary, Guatemala, Nicaragua, Scotland, Thailand, Yugoslavia, El-Salvador, Trinadad&Tobago, Peru, Hong, Holand-Netherlands.
  • income: >50K, <=50K.

Instructions

Write a C++ program that reads in this dataset, parses the fields, and stores them in a data structure(s) for easy lookup later.

You should only read the file once, and should thoughtfully choose data structures that enable you to store all of the data you need in a way that the desired values can be easily found / computed.

  1. Prompt the user for the location of the census text file.
  2. Read in the file, close it, and display the text, "Census file loaded."
  3. List the number of individuals in each occupation.
  4. Prompt the user for a "native-country", then for that country display the following:
    1. The average "education-num" or number of years in school.
    2. The number of individuals that have income >50K, and the number <=50K.
    3. The number of individuals in each marital category.
  5. Continue prompting the user for a native-country, and repeat the last step, until the user enters: "quit".

A few other guidelines / hints:

  • You may create any classes or functions that are helpful.
  • You may use any classes or functions from the Standard Template Library (STL) that are helpful. (You might want to consider a data structure that associates keys to values).
  • Don't copy the census text file to your home directory, it is large and can fill up your disk quota. If you want a smaller one to work with for testing purposes, you can use: /home/cs241/census_small.txt

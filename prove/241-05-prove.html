05 Prove : Assignment - Ecommerce Product Inventory, Part II

Overview

Having product lists and a way to compute shipping and total costs from your last project has immensely helped your growing on-line business, but to benefit even more from your program, you need to add customers.

This week, you will build on your Product library from last week, and create classes to manage orders and customers as well.

Instructions

Write a C++ program according to the following:

  1. Create the following classes (as specified below):
    • Address - Contains a customer's address
    • Customer - Contains a name and an Address
    • Product - From last week (we will add a few methods to it)
    • Order - Contains a Product, quantity, and a Customer

    2. Address Class

  2. Create a class for an Address that has the following private data members:
    • street : string
    • city : string
    • state : string
    • zip : string
    It should also contain the following public methods:
    • getStreet / setStreet
    • getCity / setCity
    • getState / setState
    • getZip / setZip
    • prompt - Prompts for each of the four values
    • display - Displays the address in the format:
      • street city, state zip
    It should also have the following constructors:
    • Default - Set the street to "unknown", the zip to "00000", and the other data members to "".
    • Non-default - Accept 4 strings (street, city, state, zip) and set each of the values.

    3. Customer Class

  3. Create a class for a Customer that has the following private data members:
    • name : string
    • address : Address
    It should also contain the following public methods:
    • getName / setName
    • getAddress / setAddress
    • prompt - Prompts for the name, then calls the address's prompt method.
    • display - Displays the name, then calls the address's display method. Thus the format is:
      • name street city, state zip
    It should also have the following constructors:
    • Default - Set the name to "unspecified" and uses the address's default constructor.
    • Non-default - Accept a name and an address object.

    4. Product Class

  4. Add the following to your Product class:
    • getName / setName
    • getDescription / setDescription
    • getBasePrice / setBasePrice
    • getWeight / setWeight
    • Default Constructor- Set the name to "none", description to "", and the weight and the base price to 0.
    • Non-default - Accepts all four of these values (name, description, basePrice, weight) and sets them.

    5. Order Class

  5. Create a class for an Order that has the following private data members:
    • product : Product
    • quantity : int
    • customer : Customer
    It should also contain the following public methods:
    • getProduct / setProduct
    • getQuantity / setQuantity
    • getCustomer / setCustomer
    • getShippingZip - Returns the Zip from the customer's address
    • getTotalPrice - Returns the total price of the product (including tax and shipping) multiplied by the quantity.
    • displayShippingLabel - Calls the customer's display method, resulting in this format:
      • name street city, state zip
    • displayReceipt - Displays the customers name followed by the displayReceipt information from the product, in this format (substituting for the correct values):
      • customerName productName Price: $ xx.xx Sales tax: $ x.xx Shipping cost: $ x.xx Total: $ xx.xx
    It should also have the following constructors:
    • Default - Set the quantity to 0 and uses the product and customer's default constructors.
    • Non-default - Accepts a Product, quantity, and Customer

    6. Main

  6. For this assignment, a main function will be provided for you. It expects the classes and methods to work as defined above. A few things in this regard:
    • You should not change main---At least for your final submission. Feel free to comment out various sections of code as you develop the program, just make sure they all make it back in for the finished product.
    • This is done to help reinforce the idea that you are making a "library" that other programmers could use. We agree upon what classes there are, and how they should interact with the outside world, but the inner details are up to you.
    • This idea of having a driver program that exercises all the methods of your class is very similar to a concept of Unit Testing, that is extremely common and important in industry.

A few other guidelines / hints:

  • Make sure to start with the files provided, and to put this assignment in its own directory.
  • Make sure your classes definitions are all in the .h files, but your method bodies are in the .cpp files. Yes, you will have a lot of files for this assignment!
  • Get started early, there are a few tricky aspects to this assignment.

Getting started

A skeleton of the files you will need to create has been provided for you. Please get it and start from there:

mkdir assign05 cd assign05 cp /home/cs241/assign05/* .
Then copy over your product class from last week:
(Assuming you are in the assign05 directory, and that last week's assignment is in the assign04 directory.) cp ../assign04/product.* .

Then don't forget to open the "makefile" and put your own name, teacher, etc. at the top.

Other Helpful Commands

13 Teach : Team Activity - Linux Shell Scripts

This activity is designed to be completed during class.

Overview

This activity will help you practice writing shell scripts in Linux.

Your assignment is to write a shell script that accepts a directory as a command line parameter, and then iterates through the .cpp files in that directory.

Instructions

Divide your teams into groups of 2-3. You should all work together on one laptop, then make sure everyone in your team gets a copy of the work that you complete.

Core Requirements

  1. Create a shell script that accepts a file as a command line parameter and then displays it to the screen as follows: (The dollar sign below just means you are typing it at the command line)

    $ ./lineCounter.sh myFile.cpp Running on file: myFile.cpp

    Make your script executable and ensure that it runs.

  2. Have your script count the number of lines in that file and display it to the screen. For example:
    3. $ ./lineCounter.sh myFile.cpp Running on file: myFile.cpp 127 myFile.cpp
  3. Change your script to accept a directory (rather than a file) as the command line parameter. Then have it iterate through all .cpp files in that directory and for each one, display the name of the file and the number of lines in that file.
    4. $ ./lineCounter.sh /home/cs241 Running on directory: /home/cs241 Files in the directory: 60 /home/cs241/check02b.cpp 47 /home/cs241/check08a.cpp 49 /home/cs241/check09a.cpp 151 /home/cs241/check09b.cpp 26 /home/cs241/check10a.cpp 212 /home/cs241/check10b.cpp 84 /home/cs241/check11a.cpp 84 /home/cs241/check11b.cpp 47 /home/cs241/check12a.cpp 21 /home/cs241/checkpoint.cpp 47 /home/cs241/ta03.cpp 47 /home/cs241/ta07.cpp 59 /home/cs241/ta09.cpp 75 /home/cs241/ta11.cpp 103 /home/cs241/ta12.cpp 29 /home/cs241/template.cpp

When you complete these requirements, show your code to your instructor to "pass it off."

Stretch Challenges

After completing the above steps. Make sure that everyone on the entire team is to this point and understands the material. Then, if you have time, move onto the following stretch challenges.

  1. Change your script so that the number of lines per file is saved to an output file, rather than to the screen. Specify the output file with another parameter (e.g., ./lineCounter.sh /home/cs241 cppLines.txt)
  2. Change your script so that it looks through sub-directories (just one level deep) for .cpp files as well.
  3. Change your script so that in additional to accepting a directory name, it also accepts a list of potential extensions (e.g., ./lineCounter.sh /home/cs241 cppLines.txt cpp h txt). Then, have it count the lines in all of those types of files.

Submission

You do not need to submit your program (just make sure that everyone has a copy of it for their reference). Instead, answer the accompanying questions in the I-Learn quiz.

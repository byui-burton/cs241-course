08 Teach : Team Activity - Inheritance

This activity is designed to be completed during class.

Overview

Create a class to represent a Circle, based on a provided Point class. It has a center (x, y), and a radius. Because a Circle has an x and a y coordinate, and more, one way to view it, would be to consider that a Circle IS-A Point. In this case, the Circle class would inherit from the Point class, and simply add a radius to it.

On the other hand, another way to view the problem is to consider that a circle HAS-A point as its center, and then also has a radius. With this approach the Circle would not inherit from the Point class, but rather, would have a member variable that is of type Point.

For this activity, we will explore both options, first focusing on the IS-A relationship, and then the HAS-A approach.

Instructions

Divide your teams into groups of 2-3. You should all work together on one laptop, then make sure everyone in your team gets a copy of the work that you complete.

Begin by copying the starting files. These files contain a working point class, skeleton files for your driver and circle class, and a makefile.

mkdir ta08_isa cd ta08_isa cp /home/cs241/ta08/* .

The provided point class has the following class diagram:

Point
-x : float
-y : float
+getX() : float
+getY() : float

+setY(float) : void

Core Requirements

  1. Create the basic Circle class as follows:

    1. Create a Circle class that inherits from, or extends, the Point class.
    2. Create a blank method for promptForCircle and display.
    3. In main, call these functions and ensure that everything compiles.

  2. Implement the promptForCircle method. It should ask for a Circle as follows:

    Enter x: 1 Enter y: 2 Enter radius: 3

    Please note that the Point class has a method called promptForPoint. Make sure you call this from your method to re-use code!

  3. Implement the display method. It should display a Circle as follows:

    (1, 2) - Radius: 3

    Please not that the Point class has a method called display. Make sure you call this from your method to re-use code! (Yes, these methods should both be called "display")

When you complete these requirements, show your code to your instructor to "pass it off"

Stretch Challenges

After completing the above steps. Make sure that everyone on the entire team is to this point and understands the material. Then, if you have time, move onto the following stretch challenges.

For the stretch challenge, you will repeat the activity using a HAS-A relationship.

  1. Create a new directory "ta08-hasa", copy the same starting files to that directory, and create a new Circle class. This time your Circle should NOT inherit from Point, but rather should have a Point as a member variable called center.

    Implement the promptForCircle and display methods for your Circle class as before.

  2. Add getters/setters for x, y, and radius. This means you need to get/set the values from the inner Pointclass, by "wrapping" them with new methods.
  3. Discuss with your team the pros and cons of the IS-A vs HAS-A approach to this problem. Determine which approach you feel is best and why. Then, include your answer and its rationale in your "Lessons Learned" question for the I-Learn assessment.

Submission

You do not need to submit your program (just make sure that everyone has a copy of it for their reference). Instead, answer the accompanying questions in the I-Learn quiz.

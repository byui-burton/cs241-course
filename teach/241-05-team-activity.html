05 Teach : Team Activity - Accessors/Mutators and Constructors

This activity is designed to be completed during class.

Overview

As a team, write a program to model a robot that can move around and fire a laser. To accomplish this you'll need to work with two classes: a Point (which has integer X and Y coordinates) and a Robot (which has a Point and an integer amount of engery).

Instructions

You will be provided with a basic version of these classes, that have the member variables, and a display function.

Here are the class diagrams of the classes according to how they are provided to you:

-y : int
+display() : void
Robot

+display() : void

Part I - Add Getters and Setters

Take the classes provided and add getters and setters for all the private data members. In particular, note the following rules

  • Our robot is limited to a grid that ranges from (1,1) to (10,10)
  • If the user tries to set X or Y to an amount less than 1, it should simply be set to 1.
  • If the user tries to set X or Y to an amount greater than 10, it should simply be set to 10.
  • The robot's engergy cannot be negative. If the user tries to set it to a negative value, it should be set to 0.

A main function has been provided for you to help test your getters and setters. Uncomment the code that is marked "Part I" and make sure it compiles and runs correctly.

Part II - Add Point Constructors

Take the classes provided and add a default constructor and a non-default constructor for the Point class. In particular, note the following rules

  • The default location of the robot is at (5, 5).
  • A non-default constructor should be created to take an X and Y coordinate, and set the values accordingly.
  • The same rules about possible values for X and Y as in the last section still apply.

A main function has been provided for you to help test the point class constructors. Uncomment the code that is marked "Part II" and make sure it compiles and runs correctly.

Part III - Add Robot Constructors

Take the classes provided and add a default constructor and a non-default constructor for the Robot class. In particular, note the following rules:

  • The default energy amount is 100.
  • A non-default constructor should be created to take an integer engergy amount, and still have the robot at its default position.
  • A non-default constructor should be created to take both a Point and an integer engergy amount.
  • The same rules about possible values for X and Y as in the last section still apply.

A main function has been provided for you to help test the point class constructors. Uncomment the code that is marked "Part III" and make sure it compiles and runs correctly.

Stretch Challenges

After completing the above steps. Make sure that everyone on the entire team is to this point and understands the material. Then, if you have time, move onto the following stretch challenges.

  1. Add "moveUp", "moveDown", "moveLeft", and "moveRight" member functions to your Robot class. Uncomment the code in Main indicated "Stretch Part I". Make sure that your Robot does not leave the range of (1, 1) to (10, 10).
  2. Add a "fireLaser" method to your class. It should output something (e.g., "Pew! Pew!") and reduce the robots energy by 25. Also, make it so that the movement methods each decrease the engergy by 10. Uncomment the code marked "Stretch Part II" and ensure that your robot never gets to negative energy.
  3. Consider that there are land mines in the following locations: (3, 3), (4, 4), and (6, 6). If the robot ever moves to those spots, reduce its energy to 0. Further, change it so that if the robot tries to move or fire a laser without enough energy, an error message should be displayed and the move or the firing should be aborted.

09 Teach : Team Activity - Polymorphism

This activity is designed to be completed during class.

Overview

BYU-Idaho is having issues with their payroll system and has asked you to create a new one. You quickly recognize that there are two types of employees, hourly employees (those that get paid a certain amount per hour) and salary employees (those that receive an annual salary).

While these two types of employees have differences, they also have a significant amount in common. So you decide to create a common base class with each of these two types as a separate derived class.

Instructions

Divide your teams into groups of 2-3. You should all work together on one laptop, then make sure everyone in your team gets a copy of the work that you complete.

For this assignment, you will start with a blank file. To keep our focus directly on the virtual functions we are working with, you may put all of our classes in the same file rather than separate .h and .cpp files.

Core Requirements

  1. Create a base class Employee, and two derived classes HourlyEmployee and SalaryEmployee, as follows:

    1. Employee should have a string name, a getter and setter for it, and a display function that displays the name.
    2. HourlyEmployee should have an int hourlyWage (and getter/setter) and should override the display function to the name and wage in the format: "John - $8/hour".
    3. SalaryEmployee should have an int salary (and getter/setter) and should override the display function to the name and salary in the format: "John - $50000/year".
    4. You may also find it beneficial to add non-default constructors to your classes, but that is up to you.

  2. Create a main function that does the following:

    1. Declare an array for 3 employees.
    2. Loop through 3 times and prompt the user for an "h" (hourly employee) or an "s" (salary employee).
    3. Then prompt for the name and the hourlyRate/salary
    4. For each employee entered, create a new employee of the correct type and add it to the array.

  3. After adding the employees to the array, have main do the following:

    1. Loop through the array and call the display method for each employee. Run your program and ensure that the correct one is called.
    2. After the display loop, free up the memory you have allocated for these objects to ensure there are no memory leaks.

When you complete these requirements, show your code to your instructor to "pass it off."

Stretch Challenges

After completing the above steps. Make sure that everyone on the entire team is to this point and understands the material. Then, if you have time, move onto the following stretch challenges.

  1. Add to your HourlyEmployee a member variable for hours. Then, add a pure virtual function to the employee class for getPayCheck() and override it in the derived classes. For hourly employees, this is calculated as the number of hours multiplied by the hourly rate. For salary employees, this should be the salary divided by 24.

    In the loop in main that calls the display function, change it to also call the getPayCheck() function and display the pay check amount.

  2. Add a regular function (not a member function of any class), "displayEmployeeData" that accepts an employee and calls its display function as well as its getPayCheck() function and displays the value. Then, remove this code from main, and replace it with a call to your new function.
  3. Change your array to a vector (a dynamically resizing array from the standard template library). Then change your loop to allow for any number of employees to be added, and stop adding them when the user enters a 'q'. Ensure that each of these employees are displayed, and cleaned up appropriately.

Submission

You do not need to submit your program (just make sure that everyone has a copy of it for their reference). Instead, answer the accompanying questions in the I-Learn quiz.

04 Teach : Team Activity - Classes

This activity is designed to be completed during class. You may finish any or all of it after class (either individually or with others) as you desire.

Overview

As a team, write a class to represent a rational number (i.e., a fraction).

Instructions

Divide your teams into groups of 2-3. You should all work together on one laptop, then make sure everyone in your team gets a copy of the work that you complete.

Core Requirements

  1. Put your class in separate files:
    • rational.h - Contains the class definition / prototypes
    • rational.cpp - Contains the bodies of the functions
    • ta04.cpp - Contains main
  2. A skeleton of these files is provided for you (plus a makefile to tell TestBed how to build it). Please create a new directory, and copy them over as follows:
    3. mkdir ta04 cd ta04 cp /home/cs241/ta04/* .
  3. Add the following data members: int top and int bottom.
  4. Add the following functions. Make sure to put their prototypes in the .h file, and their bodies in the .cpp file.
    1. prompt - Ask the user to enter the top and the bottom. Save these in the member variables.
    2. display - Display the values in the format: "top/bottom".
    3. displayDecimal - Display the value of the rational number in decimal format.
  5. In ta04.cpp, write a main function that creates a rational number, prompts the user for its values, and displays it.
  6. Compile your program using: g++ rational.cpp ta04.cpp (notice rational.h is not in that list).
  7. Create tar file for your project using: tar -cf ta04.tar makefile *.h *.cpp
  8. Run your tar file against the testBed for this activity using: testBed cs241/ta04 ta04.tar
  9. Show your code to your instructor to "pass it off"

Stretch Challenges

After completing the above steps. Make sure that everyone on the entire team is to this point and understands the material. Then, if you have time, move onto the following stretch challenges.

  1. Change your display function so that if the value is an improper fraction (larger on top than bottom), it display it as a mixed number, such as: "1 1/4", rather than "5/4".
  2. Write a multiplyBy function that accepts another rational number and changes the value of the current object (multiply the two numerators to get the new numerator and the two denominators to get the new denominator). Modify your main function to demonstrate it.
  3. Create a reduce function, that reduces rational numbers to their most basic equivalent form (e.g., 2/6 becomes 1/3). Modify your main function to demonstrate it.

Completion

By following the steps above, you should complete and report on the following:

Core Requirements:

  1. prompt
  2. display and displayDecimal
  3. Code compiles, tar file created, passed testbed

Stretch Challenges: (Make sure everyone on your team is done with their core features before working on these.)

  1. Mixed numbers
  2. multiplyBy
  3. reduce

Submission

You do not need to submit your program (just make sure that everyone has a copy of it for their reference). Instead, answer the accompanying questions in the I-Learn quiz.

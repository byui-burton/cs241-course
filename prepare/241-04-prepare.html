Please note that the textbook mentions putting your classes in separate files and using "makefiles" to compile them all together. We will put our classes in separate files as described in the textbook, however we will not learn the ins and outs of makefiles until a little later on. Instead, we will compile the separate files together by putting all the .cpp files (not the .h files) on the commandline.

As an example, consider that you had a shape class, defined in shape.cpp and shape.h and then had a main function that used it defined in driver.cpp. You would then have these files:

shape.h shape.cpp driver.cpp

You would compile and run your program with the lines:

g++ shape.cpp driver.cpp a.out

Note that we did NOT include shape.h when compiling, but that we did include all of the .cpp files.

TAR files, TestBed, and Submitting

The testBed and submit commands only work with a single file. Therefore, we need to "tar up" our files into a single .tar file (very similar to zipping them to create a zip file). To create a new tar file named check04b.tar with all of the files for this assignment, go to your directory and run the following command:

tar -cf check04b.tar makefile *.h *.cpp

This "creates a file" (hence, cf) with the name of whatever comes after it. Then comes a list of all the files to include. The line above includes a file called, "makefile" and then all the .h and .cpp files in the current directory.

Please note that you MUST include the name of a tar file, if you forget and write something like this:

THIS IS BAD: tar -cf *.cpp *.h

It will assume that one of your .cpp files is the name that you want to use for your tar file, and overwrite it. Yikes!

Once you have a tar file, you can use it with testBed or submit, just like before:

testBed cs241/check04b check04b.tar submit check04b.tar

These tools will use a makefile to tell them how to compile your program. At this point, we will provide one for you to start with. The makefile also it include your name, assignment number, etc. so be sure to fill that in.

03 Prepare : Checkpoint A - Exceptions

After completing (or while completing) the preparation material for this week, complete the following exercise.

Overview

This checkpoint is intended to help you practice the syntax of Exception Handling.

Instructions

Write a C++ program that prompts the user for a positive even number, less than 100. If the user enters a number that is not valid, an exception message should be thrown from one function, caught in another and displayed to the user. In particular

  1. Write a function, prompt, that prompts the user for a number, and returns it.
  2. Have main call this function and then display the result.
  3. In the prompt function, after getting the number from the user, the program should check for the following conditions (in this order) and throw a string with the accompanying message:
    1. Condition: Number is less than 0. Message: "The number cannot be negative."
    2. Condition: Number is greater than 100. Message: "The number cannot be greater than 100."
    3. Condition: Number is odd. Message: "The number cannot be odd."
  4. Set up main to try the function, and catch a potential exception.
  5. If the exception is thrown. Main should display: "Error: " followed by the content of the exception message that was caught. It should not try display the number returned from the prompt function in this case.

Helpful hints:

  • Remember that there is a difference between a string and a character array.
  • If you are expecting to catch a string but you throw a character array (e.g., throw "message here";) your data types will not line up and you will likely have an uncaught exception resulting in your program terminating with a message about a const char *.
  • If you are trying to catch a string, you must throw a string, which could be done like so: throw string("message here");.

Sample Output

The following is an example of output for this program:

Enter a number: 4 The number is 4.

Another run may look as follows:

Enter a number: -65 Error: The number cannot be negative.

Another run may look as follows:

Enter a number: 333 Error: The number cannot be greater than 100.

Another run may look as follows:

Enter a number: 17 Error: The number cannot be odd.
